The portal system was not static, in fact, Datum is a VR game project where the player can summon portals at his hands position in order to change the path of incoming objects. And hand can move all along of the game so
So I used some 3d vector maths to calculate the point of intersection of a portal based on the portal plan and the velocity vector of incoming objects.
Then I’m using the distance from the portal centre to know if the object passes throw or not. And finally, if the object velocity / portal forward vector dot product tells me which way goes the object, I teleport the object based on that and in order to make it fancy I made a little shader that cut the object at the portal intersection.
Voilà, there is much more that I did in order to archive the final result, but the main concept is here.
